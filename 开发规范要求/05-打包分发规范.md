## 2. 端口配置规范

### 2.1 端口分配（8500-8600 范围）

| 服务                 | 默认端口  | 配置项              | 说明             | 协议      |
| -------------------- | --------- | ------------------- | ---------------- | --------- |
| **API Server**       | 8501      | `API_PORT`          | FastAPI 主服务   | HTTP/REST |
| **Ollama Proxy**     | 11434     | `OLLAMA_PROXY_PORT` | Ollama API 代理  | HTTP/REST |
| **WebSocket**        | 8502      | `WS_PORT`           | 实时通信服务     | WebSocket |
| **Subtitle Service** | 8503      | `SUBTITLE_PORT`     | 字幕窗口通信     | HTTP/IPC  |
| **TTS Service**      | 8504      | `TTS_PORT`          | 语音合成服务     | HTTP/REST |
| **Log Service**      | 8505      | `LOG_PORT`          | 独立日志服务     | HTTP/SSE  |
| **预留端口 1**       | 8506      | -                   | 未来扩展         | -         |
| **预留端口 2**       | 8507      | -                   | 未来扩展         | -         |
| **...**              | 8508-8599 | -                   | 预留扩展         | -         |
| **动态端口池**       | 8600      | `DYNAMIC_PORT_BASE` | 动态分配起始端口 | -         |

> **注意**: Log Service (8505) 是独立微服务，需要与 API Server (8501) 同时启动。主服务通过 HTTP API 将日志上报到日志服务。

### 2.2 端口冲突处理策略

```python
# backend/src/core/port_manager.py
import socket
from typing import Optional

class PortManager:
    """端口管理器，处理端口分配和冲突"""

    DEFAULT_PORTS = {
        'api': 8500,
        'ollama_proxy': 8501,
        'websocket': 8502,
        'subtitle': 8503,
        'tts': 8504,
        'log': 8505,
    }

    PORT_RANGE = (8500, 8600)

    @classmethod
    def is_port_available(cls, port: int) -> bool:
        """检查端口是否可用"""
        try:
            with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
                s.bind(('127.0.0.1', port))
                return True
        except OSError:
            return False

    @classmethod
    def find_available_port(cls, preferred_port: int) -> int:
        """查找可用端口，优先使用指定端口"""
        if cls.is_port_available(preferred_port):
            return preferred_port

        # 在范围内查找下一个可用端口
        for port in range(preferred_port + 1, cls.PORT_RANGE[1] + 1):
            if cls.is_port_available(port):
                return port

        raise RuntimeError(f"无法在范围 {cls.PORT_RANGE} 内找到可用端口")

    @classmethod
    def get_service_port(cls, service_name: str, config: Optional[dict] = None) -> int:
        """获取服务端口，支持配置覆盖"""
        preferred = config.get(f'{service_name}_port') if config else None
        preferred = preferred or cls.DEFAULT_PORTS.get(service_name, 8500)
        return cls.find_available_port(preferred)
```

## 3. Python 后端打包方案

### 3.1 方案对比

| 方案                | 启动速度 | 体积 | 安全性 | 复杂度 | 推荐度     |
| ------------------- | -------- | ---- | ------ | ------ | ---------- |
| **PyInstaller**     | 中等     | 大   | 低     | 低     | ⭐⭐⭐     |
| **Nuitka**          | 快       | 中   | 高     | 中     | ⭐⭐⭐⭐⭐ |
| **embedded Python** | 快       | 大   | 中     | 高     | ⭐⭐⭐⭐   |
| **PyOxidizer**      | 快       | 小   | 高     | 高     | ⭐⭐⭐⭐   |
| **Rust + Tauri**    | 快       | 小   | 高     | 中     | ⭐⭐⭐⭐⭐ |

> **推荐**: 使用 **Rust + Tauri** 框架构建桌面应用，替代 Electron。Tauri 提供更小的包体积、更好的性能和更强的安全性。

### 3.3 打包脚本

```python
# scripts/build_backend.py
#!/usr/bin/env python3
"""
后端打包脚本
"""
import os
import sys
import shutil
import subprocess
from pathlib import Path

# 端口配置检查
REQUIRED_PORTS = [8500, 8501, 8502, 8503, 8504, 8505]

def check_ports():
    """检查端口配置是否正确"""
    # 检查主服务配置
    main_config = Path("backend/src/core/config.py")
    if main_config.exists():
        content = main_config.read_text(encoding='utf-8')
        for port in REQUIRED_PORTS:
            if str(port) not in content:
                print(f"⚠️ 警告: 端口 {port} 未在主服务配置中找到")
                return False

    # 检查日志服务配置
    log_config = Path("log_service/src/core/database.py")
    if log_config.exists():
        content = log_config.read_text(encoding='utf-8')
        if "8505" not in content:
            print("⚠️ 警告: 日志服务端口 8505 未在配置中找到")
            return False

    print("✅ 端口配置检查通过")
    return True

def build_service(service_dir: str, output_name: str, output_dir: str):
    """构建单个服务"""
    print(f"🔨 构建 {output_name}...")

    cmd = [
        sys.executable, "-m", "nuitka",
        "--standalone",
        "--enable-plugin=anti-bloat",
        "--include-package=fastapi",
        "--include-package=uvicorn",
        "--include-package=pydantic",
        "--include-package=aiosqlite",
        "--windows-disable-console",
        "--windows-icon-from-ico=resources/icon.ico",
        "--output-dir", output_dir,
        f"{service_dir}/src/main.py"
    ]

    subprocess.run(cmd, check=True)
    print(f"✅ {output_name} 构建完成")

def build_windows():
    """构建 Windows 版本（主服务 + 日志服务）"""
    print("🔨 构建 Windows 版本...")

    # 构建主服务
    build_service("backend", "主服务 (API Server)", "dist/windows/backend")

    # 构建日志服务
    build_service("log_service", "日志服务 (Log Service)", "dist/windows/log_service")

    print("✅ Windows 构建完成")

def build_macos():
    """构建 macOS 版本（主服务 + 日志服务）"""
    print("🔨 构建 macOS 版本...")

    # 构建主服务
    build_service("backend", "主服务 (API Server)", "dist/macos/backend")

    # 构建日志服务
    build_service("log_service", "日志服务 (Log Service)", "dist/macos/log_service")

    print("✅ macOS 构建完成")

def build_linux():
    """构建 Linux 版本（主服务 + 日志服务）"""
    print("🔨 构建 Linux 版本...")

    # 构建主服务
    build_service("backend", "主服务 (API Server)", "dist/linux/backend")

    # 构建日志服务
    build_service("log_service", "日志服务 (Log Service)", "dist/linux/log_service")

    print("✅ Linux 构建完成")

if __name__ == "__main__":
    if not check_ports():
        sys.exit(1)

    platform = sys.platform
    if platform == "win32":
        build_windows()
    elif platform == "darwin":
        build_macos()
    else:
        build_linux()
```

### 4.2 主进程端口管理

```typescript
// frontend/electron/main.ts
import { app, ipcMain } from "electron";
import { spawn, ChildProcess } from "child_process";
import * as net from "net";

// 端口配置
const PORT_CONFIG = {
  API: 8500,
  OLLAMA_PROXY: 8501,
  WEBSOCKET: 8502,
  SUBTITLE: 8503,
  TTS: 8504,
  LOG: 8505,
};

class PortManager {
  private usedPorts: Map<string, number> = new Map();

  async findAvailablePort(preferredPort: number): Promise<number> {
    return new Promise((resolve, reject) => {
      const server = net.createServer();

      server.listen(preferredPort, "127.0.0.1", () => {
        const port = (server.address() as net.AddressInfo).port;
        server.close(() => resolve(port));
      });

      server.on("error", (err: any) => {
        if (err.code === "EADDRINUSE") {
          // 端口被占用，尝试下一个
          this.findAvailablePort(preferredPort + 1)
            .then(resolve)
            .catch(reject);
        } else {
          reject(err);
        }
      });
    });
  }

  async allocatePorts(): Promise<Record<string, number>> {
    const ports: Record<string, number> = {};

    for (const [name, defaultPort] of Object.entries(PORT_CONFIG)) {
      ports[name] = await this.findAvailablePort(defaultPort);
      this.usedPorts.set(name, ports[name]);
      console.log(`✅ 分配端口 ${name}: ${ports[name]}`);
    }

    return ports;
  }

  getPort(serviceName: string): number | undefined {
    return this.usedPorts.get(serviceName);
  }
}

// 后端进程管理
class BackendManager {
  private mainProcess: ChildProcess | null = null;
  private logProcess: ChildProcess | null = null;
  private portManager = new PortManager();

  async start() {
    const ports = await this.portManager.allocatePorts();

    // 设置环境变量
    const env = {
      ...process.env,
      AIEFFECT_API_PORT: String(ports.API),
      AIEFFECT_OLLAMA_PROXY_PORT: String(ports.OLLAMA_PROXY),
      AIEFFECT_WS_PORT: String(ports.WEBSOCKET),
      AIEFFECT_SUBTITLE_PORT: String(ports.SUBTITLE),
      AIEFFECT_TTS_PORT: String(ports.TTS),
      AIEFFECT_LOG_PORT: String(ports.LOG),
    };

    // 1. 先启动日志服务 (必须先启动，因为主服务依赖它)
    const logServicePath = app.isPackaged
      ? path.join(process.resourcesPath, "log_service", "log-service")
      : path.join(__dirname, "../../log_service/dist/log-service");

    this.logProcess = spawn(logServicePath, [], {
      env: { ...env, PORT: String(ports.LOG) },
      detached: false,
    });

    this.logProcess.stdout?.on("data", (data) => {
      console.log(`[LogService] ${data}`);
    });

    this.logProcess.stderr?.on("data", (data) => {
      console.error(`[LogService Error] ${data}`);
    });

    // 等待日志服务启动
    await new Promise((resolve) => setTimeout(resolve, 2000));
    console.log("✅ 日志服务已启动");

    // 2. 启动主服务
    const backendPath = app.isPackaged
      ? path.join(process.resourcesPath, "backend", "aieffect-backend")
      : path.join(__dirname, "../../backend/dist/aieffect-backend");

    this.mainProcess = spawn(backendPath, [], {
      env,
      detached: false,
    });

    this.mainProcess.stdout?.on("data", (data) => {
      console.log(`[Backend] ${data}`);
    });

    this.mainProcess.stderr?.on("data", (data) => {
      console.error(`[Backend Error] ${data}`);
    });

    console.log("✅ 主服务已启动");
    return ports;
  }

  stop() {
    // 先停止主服务
    if (this.mainProcess) {
      this.mainProcess.kill();
      this.mainProcess = null;
      console.log("🛑 主服务已停止");
    }

    // 再停止日志服务
    if (this.logProcess) {
      this.logProcess.kill();
      this.logProcess = null;
      console.log("🛑 日志服务已停止");
    }
  }
}

export const backendManager = new BackendManager();
```

### 4.3.2 Rust 服务管理代码

```rust
// src-tauri/src/services.rs
use std::process::{Child, Command};
use std::sync::Mutex;
use tauri::api::process::Command as TauriCommand;

pub struct ServiceManager {
    main_service: Mutex<Option<Child>>,
    log_service: Mutex<Option<Child>>,
}

impl ServiceManager {
    pub fn new() -> Self {
        ServiceManager {
            main_service: Mutex::new(None),
            log_service: Mutex::new(None),
        }
    }

    pub fn start_services(&self) -> Result<(), String> {
        let resource_dir = tauri::api::path::resource_dir()
            .ok_or("无法获取资源目录")?;

        // 1. 启动日志服务
        let log_service_path = resource_dir.join("bin/log_service");
        let log_child = Command::new(log_service_path)
            .env("PORT", "8505")
            .spawn()
            .map_err(|e| format!("启动日志服务失败: {}", e))?;

        *self.log_service.lock().unwrap() = Some(log_child);
        println!("✅ 日志服务已启动");

        // 等待日志服务就绪
        std::thread::sleep(std::time::Duration::from_secs(2));

        // 2. 启动主服务
        let main_service_path = resource_dir.join("bin/backend");
        let main_child = Command::new(main_service_path)
            .env("API_PORT", "8501")
            .env("LOG_PORT", "8505")
            .spawn()
            .map_err(|e| format!("启动主服务失败: {}", e))?;

        *self.main_service.lock().unwrap() = Some(main_child);
        println!("✅ 主服务已启动");

        Ok(())
    }

    pub fn stop_services(&self) {
        // 先停止主服务
        if let Some(mut child) = self.main_service.lock().unwrap().take() {
            let _ = child.kill();
            println!("🛑 主服务已停止");
        }

        // 再停止日志服务
        if let Some(mut child) = self.log_service.lock().unwrap().take() {
            let _ = child.kill();
            println!("🛑 日志服务已停止");
        }
    }
}

// Tauri 命令
#[tauri::command]
fn start_backend_services(state: tauri::State<ServiceManager>) -> Result<String, String> {
    state.start_services()?;
    Ok("服务已启动".to_string())
}

#[tauri::command]
fn stop_backend_services(state: tauri::State<ServiceManager>) {
    state.stop_services();
}
```

### 5.1 Windows 代码签名

```powershell
# scripts/sign-windows.ps1
param(
    [string]$CertificatePath,
    [string]$CertificatePassword,
    [string]$ExecutablePath
)

# 使用 signtool 签名
$signtool = "C:\Program Files (x86)\Windows Kits\10\bin\10.0.19041.0\x64\signtool.exe"

& $signtool sign `
    /f $CertificatePath `
    /p $CertificatePassword `
    /tr http://timestamp.digicert.com `
    /td sha256 `
    /fd sha256 `
    /a `
    $ExecutablePath

Write-Host "✅ Windows 签名完成: $ExecutablePath"
```

### 5.2 macOS 代码签名与公证

```bash
#!/bin/bash
# scripts/sign-macos.sh

APP_NAME="AIEffect"
APP_PATH="dist/mac/${APP_NAME}.app"
DMG_PATH="dist/${APP_NAME}.dmg"
DEVELOPER_ID="Developer ID Application: Your Name (TEAM_ID)"

APP_SPECIFIC_PASSWORD="xxxx-xxxx-xxxx-xxxx"
TEAM_ID="XXXXXXXXXX"

# 签名应用
echo "🔏 签名应用..."
codesign --deep --force --verify --verbose \
    --sign "$DEVELOPER_ID" \
    --options runtime \
    "$APP_PATH"

# 创建 DMG
echo "📦 创建 DMG..."
# ... DMG 创建命令 ...

# 签名 DMG
codesign --force --sign "$DEVELOPER_ID" "$DMG_PATH"

# 公证
echo "📤 提交公证..."
xcrun notarytool submit "$DMG_PATH" \
    --apple-id "$APPLE_ID" \
    --password "$APP_SPECIFIC_PASSWORD" \
    --team-id "$TEAM_ID" \
    --wait

#  staple
echo "📌 Staple..."
xcrun stapler staple "$DMG_PATH"

echo "✅ macOS 签名和公证完成"
```

## 6. 自动更新配置

### 6.1 更新服务器配置

```typescript
// frontend/electron/updater.ts
import { autoUpdater } from "electron-updater";
import { dialog, ipcMain } from "electron";
import * as path from "path";

// 配置自动更新
autoUpdater.autoDownload = false;
autoUpdater.autoInstallOnAppQuit = true;

// 检查更新
export function checkForUpdates() {
  autoUpdater.checkForUpdatesAndNotify();
}

// 监听更新事件
autoUpdater.on("checking-for-update", () => {
  console.log("🔍 检查更新...");
});

autoUpdater.on("update-available", (info) => {
  console.log("📦 发现新版本:", info.version);

  dialog
    .showMessageBox({
      type: "info",
      title: "发现新版本",
      message: `AIEffect ${info.version} 可用`,
      detail: "是否现在下载更新？",
      buttons: ["立即下载", "稍后提醒"],
      defaultId: 0,
    })
    .then((result) => {
      if (result.response === 0) {
        autoUpdater.downloadUpdate();
      }
    });
});

autoUpdater.on("update-not-available", () => {
  console.log("✅ 已是最新版本");
});

autoUpdater.on("download-progress", (progress) => {
  const percent = Math.round(progress.percent);
  console.log(`⬇️ 下载进度: ${percent}%`);

  // 发送进度到渲染进程
  mainWindow?.webContents.send("update-progress", {
    percent,
    speed: progress.bytesPerSecond,
  });
});

autoUpdater.on("update-downloaded", (info) => {
  console.log("✅ 更新下载完成");

  dialog
    .showMessageBox({
      type: "info",
      title: "更新已就绪",
      message: "新版本已下载完成",
      detail: "重启应用以安装更新",
      buttons: ["立即重启", "稍后重启"],
      defaultId: 0,
    })
    .then((result) => {
      if (result.response === 0) {
        autoUpdater.quitAndInstall();
      }
    });
});

autoUpdater.on("error", (error) => {
  console.error("❌ 更新错误:", error);
});
```

## 11. 微服务架构说明

### 11.1 服务依赖关系

```
┌─────────────────────────────────────────────────────────────┐
│                     AIEffect Desktop App                     │
│  ┌─────────────────┐      ┌─────────────────┐              │
│  │   Frontend      │      │   Rust/Tauri    │              │
│  │   (React)       │◄────►│   (Shell)       │              │
│  └─────────────────┘      └────────┬────────┘              │
└────────────────────────────────────┼────────────────────────┘
                                     │
                    ┌────────────────┼────────────────┐
                    │                │                │
                    ▼                ▼                ▼
           ┌────────────────┐ ┌────────────────┐ ┌──────────────┐
           │  Main Service  │ │  Log Service   │ │  Ollama      │
           │  (Port 8501)   │ │  (Port 8505)   │ │  (Port 11434)│
           │                │ │                │ │              │
           │ - API Routes   │ │ - Log Collect  │ │ - LLM API    │
           │ - Business     │ │ - Log Query    │ │              │
           │ - Proxy Layer  │ │ - SSE Stream   │ │              │
           └───────┬────────┘ └───────┬────────┘ └──────────────┘
                   │                  │
                   │     HTTP API     │
                   └─────────────────►│
                   (Log Ingest)       │
                                      ▼
                             ┌────────────────┐
                             │  SQLite DB     │
                             │  (logs.db)     │
                             └────────────────┘
```

### 11.2 服务启动顺序

1. **日志服务 (Port 8505)** - 必须先启动
   - 提供日志收集 API
   - 提供 SSE 实时流
   - 存储日志到 SQLite

2. **主服务 (Port 8501)** - 依赖日志服务
   - 通过 RemoteLogHandler 上报日志
   - API 代理到日志服务
   - 提供业务功能

### 11.3 关键设计决策

| 决策             | 说明                                               |
| ---------------- | -------------------------------------------------- |
| **独立日志服务** | 日志服务独立运行，避免影响主服务性能               |
| **HTTP 上报**    | 使用 HTTP API 而非共享数据库，实现真正的微服务解耦 |
| **批量上报**     | RemoteLogHandler 批量上报日志，减少网络开销        |
| **自动降级**     | 日志服务不可用时，自动降级到本地文件               |
| **向后兼容**     | 主服务保留原有 API，通过代理层转发到日志服务       |

### 11.4 打包注意事项

1. **必须同时打包两个服务**
   - `backend/` → `bin/backend`
   - `log_service/` → `bin/log_service`

2. **启动顺序**
   - 先启动日志服务
   - 等待 2 秒确保就绪
   - 再启动主服务

3. **数据目录分离**
   - 主服务数据: `data/`
   - 日志服务数据: `log_data/`

4. **端口管理**
   - 使用 PortManager 动态分配端口
   - 通过环境变量传递端口配置
   - 处理端口冲突（自动递增）
