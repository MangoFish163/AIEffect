# AIEffect 独立日志服务设计文档

## 1. 设计概述

### 1.1 设计目标

将日志服务从单体 API Gateway 中拆分，形成独立的微服务，实现：

- **服务解耦**：日志服务独立部署、独立扩容
- **性能隔离**：大量日志写入不影响主服务性能
- **专业优化**：针对日志场景进行专门优化（压缩、归档、冷热分离）
- **水平扩展**：支持多实例部署和负载均衡

### 1.2 架构演进

```
当前架构（单体）                    目标架构（日志独立）
┌─────────────────────┐            ┌─────────────────────┐  ┌─────────────────────┐
│   API Gateway       │            │   API Gateway       │  │   Log Service       │
│   (Port 8501)       │            │   (Port 8501)       │  │   (Port 8505)       │
│  ┌───────────────┐  │            │  ┌───────────────┐  │  │  ┌───────────────┐  │
│  │ /api/logs/*   │  │     →      │  │ 日志上报接口   │  │  │  │ SSE Stream    │  │

│  └───────────────┘  │            └─────────────────────┘  │  └───────────────┘  │
└─────────────────────┘                      │                └─────────────────────┘
                                             │                          ▲
                                             └──────────── HTTP ────────┘
```

## 2. 与现有数据库设计的对比分析

### 2.1 现有 system_logs 表结构

当前 `数据库设计.md` 中定义的日志表：

```sql
CREATE TABLE system_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    level TEXT NOT NULL,                 -- 级别: DEBUG/INFO/WARN/ERROR/SUCCESS
    module TEXT,                         -- 模块名称
    message TEXT NOT NULL,               -- 日志消息
    metadata TEXT,                       -- 元数据 (JSON)
    source_file TEXT,                    -- 源文件
    source_line INTEGER,                 -- 行号
    exception_type TEXT,                 -- 异常类型
    exception_traceback TEXT             -- 异常堆栈
);

-- 索引
CREATE INDEX idx_system_logs_timestamp ON system_logs(timestamp DESC);
CREATE INDEX idx_system_logs_level ON system_logs(level);
CREATE INDEX idx_system_logs_module ON system_logs(module);

-- FTS5 全文搜索
CREATE VIRTUAL TABLE IF NOT EXISTS system_logs_fts USING fts5(
    message,
    content='system_logs',
    content_rowid='id'
);
```

### 2.2 冲突点分析

| 冲突项         | 现有设计               | 独立日志服务设计                           | 取舍决策                    |
| -------------- | ---------------------- | ------------------------------------------ | --------------------------- |
| **数据库位置** | 主数据库 `aieffect.db` | 独立数据库 `log_service.db`                | ✅ 独立数据库，避免主库膨胀 |
| **表名**       | `system_logs`          | `logs_YYYY_MM`（按月分表）                 | ✅ 分表设计，便于归档清理   |
| **存储字段**   | 基础字段               | 增加 `process_id`, `thread_id`, `trace_id` | ✅ 增加分布式追踪字段       |
| **全文搜索**   | FTS5 虚拟表            | 保留 FTS5 + 可选 Elasticsearch             | ✅ 保留 FTS5，大规模时扩展  |
| **数据保留**   | 无自动清理策略         | 自动归档 + TTL 清理                        | ✅ 独立服务需自管理生命周期 |
| **告警功能**   | 无                     | 增加告警规则表                             | ✅ 独立服务增加告警能力     |

## 3. 日志服务数据库设计

### 3.1 数据库选型

- **数据库**: SQLite 3（轻量级，适合桌面应用）
- **文件位置**: `data/log_service.db`
- **分表策略**: 按月自动分表（`logs_2024_01`, `logs_2024_02`...）
- **归档策略**: 3个月前的数据自动归档到压缩文件

### 3.2 核心表结构

#### 3.2.1 日志主表（按月分表）

```sql
-- 动态创建的月度日志表: logs_2024_01, logs_2024_02, ...
CREATE TABLE logs_YYYY_MM (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME NOT NULL,         -- 日志时间（带毫秒精度）
    level VARCHAR(10) NOT NULL,          -- DEBUG, INFO, WARNING, ERROR, CRITICAL
    module VARCHAR(100) NOT NULL,        -- 模块名（如: app.api.proxy）
    message TEXT NOT NULL,               -- 日志消息

    -- 源代码位置
    source_file VARCHAR(255),            -- 源文件路径
    source_line INTEGER,                 -- 行号
    function_name VARCHAR(100),          -- 函数名

    -- 进程信息
    process_id INTEGER,                  -- 进程ID
    thread_id INTEGER,                   -- 线程ID

    -- 分布式追踪（未来扩展）
    trace_id VARCHAR(32),                -- 追踪ID
    span_id VARCHAR(16),                 -- 跨度ID
    parent_span_id VARCHAR(16),          -- 父跨度ID

    -- 异常信息
    exception_type VARCHAR(100),         -- 异常类型
    exception_message TEXT,              -- 异常消息
    exception_traceback TEXT,            -- 异常堆栈

    -- 扩展元数据
    metadata JSON,                       -- 扩展字段（JSON格式）

    -- 服务端记录时间
    ingested_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    -- 分片键（用于水平扩展）
    shard_key INTEGER GENERATED ALWAYS AS (
        CAST(strftime('%s', timestamp) AS INTEGER) % 16
    ) STORED
);

-- 索引设计（覆盖常见查询场景）
CREATE INDEX idx_logs_time ON logs_YYYY_MM(timestamp DESC);
CREATE INDEX idx_logs_level ON logs_YYYY_MM(level);
CREATE INDEX idx_logs_module ON logs_YYYY_MM(module);
CREATE INDEX idx_logs_time_level ON logs_YYYY_MM(timestamp, level);
CREATE INDEX idx_logs_time_module ON logs_YYYY_MM(timestamp, module);
CREATE INDEX idx_logs_trace ON logs_YYYY_MM(trace_id) WHERE trace_id IS NOT NULL;
```

#### 3.2.2 全文搜索虚拟表

```sql
-- FTS5 全文搜索表（与日志主表关联）
CREATE VIRTUAL TABLE logs_fts USING fts5(
    message,                             -- 搜索内容
    module,                              -- 模块名（加权搜索）
    exception_message,                   -- 异常消息
    content='logs_YYYY_MM',              -- 关联的日志表
    content_rowid='id'
);

-- 自动同步触发器
CREATE TRIGGER tr_logs_fts_insert AFTER INSERT ON logs_YYYY_MM
BEGIN
    INSERT INTO logs_fts(rowid, message, module, exception_message)
    VALUES (NEW.id, NEW.message, NEW.module, NEW.exception_message);
END;

CREATE TRIGGER tr_logs_fts_delete AFTER DELETE ON logs_YYYY_MM
BEGIN
    INSERT INTO logs_fts(logs_fts, rowid, message, module, exception_message)
    VALUES ('delete', OLD.id, OLD.message, OLD.module, OLD.exception_message);
END;
```

#### 3.2.3 预聚合统计表

```sql
-- 小时级统计（加速仪表盘查询）
CREATE TABLE log_stats_hourly (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    hour DATETIME NOT NULL,              -- 整点时间
    level VARCHAR(10) NOT NULL,          -- 日志级别
    module VARCHAR(100),                 -- 模块名（可选，NULL表示全部）
    count INTEGER DEFAULT 0,             -- 日志条数
    error_count INTEGER DEFAULT 0,       -- 错误数
    avg_message_length REAL,             -- 平均消息长度

    UNIQUE(hour, level, module)
);

CREATE INDEX idx_stats_hourly_time ON log_stats_hourly(hour DESC);
CREATE INDEX idx_stats_hourly_level ON log_stats_hourly(level);

-- 日级统计
CREATE TABLE log_stats_daily (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date DATE NOT NULL,                  -- 日期
    level VARCHAR(10) NOT NULL,
    module VARCHAR(100),
    count INTEGER DEFAULT 0,
    error_count INTEGER DEFAULT 0,

    UNIQUE(date, level, module)
);
```

#### 3.2.4 告警规则表

```sql
CREATE TABLE alert_rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,          -- 规则名称
    description TEXT,                    -- 规则描述
    enabled BOOLEAN DEFAULT 1,           -- 是否启用

    -- 匹配条件
    level_min VARCHAR(10),               -- 最低级别（如: ERROR）
    module_pattern VARCHAR(100),         -- 模块匹配（支持通配符 *）
    message_pattern VARCHAR(255),        -- 消息匹配（支持 LIKE）

    -- 触发条件
    condition_type VARCHAR(20) NOT NULL, -- threshold, rate, pattern
    threshold_count INTEGER,             -- 阈值次数
    time_window INTEGER,                 -- 时间窗口（秒）

    -- 通知配置
    notify_type VARCHAR(20),             -- webhook, sse, log
    notify_config JSON,                  -- 通知方式配置

    -- 冷却配置
    cooldown_seconds INTEGER DEFAULT 300, -- 冷却时间（避免频繁告警）
    last_triggered_at DATETIME,          -- 上次触发时间

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_alert_rules_enabled ON alert_rules(enabled);
```

#### 3.2.5 告警历史表

```sql
CREATE TABLE alert_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    rule_id INTEGER NOT NULL,            -- 关联的规则ID
    triggered_at DATETIME NOT NULL,      -- 触发时间
    resolved_at DATETIME,                -- 恢复时间
    severity VARCHAR(10),                -- 告警级别: info, warning, critical
    message TEXT,                        -- 告警消息
    context JSON,                        -- 触发时的上下文（相关日志ID列表）
    notified BOOLEAN DEFAULT 0,          -- 是否已通知

    FOREIGN KEY (rule_id) REFERENCES alert_rules(id)
);

CREATE INDEX idx_alert_history_rule ON alert_history(rule_id);
CREATE INDEX idx_alert_history_triggered ON alert_history(triggered_at DESC);
```

#### 3.2.6 归档记录表

```sql
CREATE TABLE log_archives (
```

#### 3.2.7 SSE 订阅会话表


### 3.3 数据库初始化脚本

```sql
-- 启用外键约束
PRAGMA foreign_keys = ON;

-- 启用 WAL 模式（提高并发性能）
PRAGMA journal_mode = WAL;
```

### 5.2 RemoteLogHandler 实现

```python
# backend/src/core/log_handler.py
import asyncio
import aiohttp
from logging import Handler, LogRecord
from typing import Optional
from datetime import datetime

class RemoteLogHandler(Handler):
    """
    远程日志处理器
    - 异步批量上报日志到独立日志服务
    - 本地队列缓冲，防止网络波动
    - 失败重试机制
    """

    def __init__(
        self,
        service_url: str = "http://localhost:8505",
        batch_size: int = 100,
        flush_interval: float = 5.0,
        max_queue_size: int = 10000,
        max_retries: int = 3
    ):
        super().__init__()
        self.service_url = service_url
        self.batch_size = batch_size
        self.flush_interval = flush_interval
        self.max_queue_size = max_queue_size
        self.max_retries = max_retries

        self._queue: asyncio.Queue = asyncio.Queue(maxsize=max_queue_size)
        self._session: Optional[aiohttp.ClientSession] = None
        self._flush_task: Optional[asyncio.Task] = None
        self._running = False
        self._retry_count: Dict[str, int] = {}

    async def start(self):
        """启动日志处理器"""
        self._session = aiohttp.ClientSession()
        self._running = True
        self._flush_task = asyncio.create_task(self._flush_loop())
        print(f"📝 RemoteLogHandler started, targeting {self.service_url}")

    async def stop(self):
        """停止日志处理器"""
        self._running = False
        #  flush remaining logs
        await self._flush_batch()
        
        if self._flush_task:
            self._flush_task.cancel()
            try:
                await self._flush_task
            except asyncio.CancelledError:
                pass
                
        if self._session:
            await self._session.close()
            
    def emit(self, record: LogRecord):
        """处理日志记录"""
        try:
            log_entry = self._format_log(record)
            # 使用 call_soon_threadsafe 确保线程安全
            asyncio.get_event_loop().call_soon_threadsafe(
                self._queue.put_nowait, log_entry
            )
        except asyncio.QueueFull:
            # 队列满时丢弃最旧的日志
            try:
                self._queue.get_nowait()
                self._queue.put_nowait(log_entry)
            except asyncio.QueueEmpty:
                pass
        except Exception:
            self.handleError(record)
            
    def _format_log(self, record: LogRecord) -> Dict[str, Any]:
        """格式化日志记录为字典"""
        log_entry = {
            "timestamp": datetime.utcnow().isoformat(),
            "level": record.levelname,
            "module": record.name,
            "message": self.format(record),
            "source_file": record.pathname,
            "source_line": record.lineno,
            "function_name": record.funcName,
            "process_id": record.process,
            "thread_id": record.thread,
        }
        
        # 异常信息
        if record.exc_info:
            log_entry["exception_type"] = record.exc_info[0].__name__ if record.exc_info[0] else None
            log_entry["exception_message"] = str(record.exc_info[1]) if record.exc_info[1] else None
            import traceback
            log_entry["exception_traceback"] = traceback.format_exception(*record.exc_info)
            
        # 额外属性
        if hasattr(record, "trace_id"):
            log_entry["trace_id"] = record.trace_id
        if hasattr(record, "metadata"):
            log_entry["metadata"] = record.metadata
            
        return log_entry
        
    async def _flush_loop(self):
        """定期刷新循环"""
        while self._running:
            try:
                await asyncio.sleep(self.flush_interval)
                await self._flush_batch()
            except asyncio.CancelledError:
                break
            except Exception as e:
                print(f"Flush loop error: {e}")
                
    async def _flush_batch(self):
        """批量上报日志"""
        if not self._session:
            return
            
        batch = []
        while len(batch) < self.batch_size and not self._queue.empty():
            try:
                batch.append(self._queue.get_nowait())
            except asyncio.QueueEmpty:
                break
                
        if not batch:
            return
            
        # 构建请求
        payload = {
            "logs": batch,
            "source": f"pid_{os.getpid()}",
            "batch_id": f"batch_{datetime.utcnow().timestamp()}"
        }
        
        # 发送请求（带重试）
        for attempt in range(self.max_retries):
            try:
                async with self._session.post(
                    f"{self.service_url}/ingest",
                    json=payload,
                    timeout=aiohttp.ClientTimeout(total=30)
                ) as response:
                    if response.status == 200:
                        return
                    else:
                        raise Exception(f"HTTP {response.status}")
            except Exception as e:
                if attempt < self.max_retries - 1:
                    await asyncio.sleep(2 ** attempt)  # 指数退避
                else:
                    print(f"Failed to send logs after {self.max_retries} retries: {e}")
                    # 最终失败，可选择写入本地文件备份
                    await self._backup_logs(batch)
                    
    async def _backup_logs(self, logs: List[Dict[str, Any]]):
        """备份失败的日志到本地文件"""
        backup_file = Path("data/log_backup.jsonl")
        backup_file.parent.mkdir(parents=True, exist_ok=True)
        
        with open(backup_file, "a", encoding="utf-8") as f:
            for log in logs:
                f.write(json.dumps(log, ensure_ascii=False) + "\n")
```

```python
# backend/src/api/logs.py - 修改后（代理模式）
from fastapi import APIRouter, HTTPException
from fastapi.responses import StreamingResponse
import httpx

router = APIRouter(prefix="/api/logs", tags=["logs"])

LOG_SERVICE_URL = "http://localhost:8505"
http_client = httpx.AsyncClient(base_url=LOG_SERVICE_URL, timeout=30.0)

@router.get("")
async def get_logs_proxy(
    level: Optional[str] = None,
    module: Optional[str] = None,
    search: Optional[str] = None,
    page: int = 1,
    page_size: int = 50
):
    """代理到日志服务（兼容旧接口）"""
    params = {"page": page, "page_size": page_size}
    if level:
        params["level"] = level
    if module:
        params["module"] = module
    if search:
        params["search"] = search

    try:
        response = await http_client.get("/logs", params=params)
        return response.json()
    except httpx.ConnectError:
        raise HTTPException(status_code=503, detail="Log service unavailable")

@router.get("/stream")
async def stream_logs_proxy():
    """代理 SSE 流"""
    async def event_generator():
        try:
            async with http_client.stream(
                "GET",
                "/stream",
                headers={"Accept": "text/event-stream"}
            ) as response:
                async for chunk in response.aiter_text():
                    yield chunk
        except httpx.ConnectError:
            yield "data: {\"error\": \"Log service unavailable\"}\n\n"

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream"
    )
```

- **WAL模式**: SQLite WAL模式提高并发

- **FTS5全文搜索**: 消息内容快速检索
- **分页查询**: 避免大结果集

### 8.3 存储优化

- **按月分表**: 便于归档和清理
- **自动归档**: 3个月前的数据自动归档
