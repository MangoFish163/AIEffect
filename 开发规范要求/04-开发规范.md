# AIEffect - 开发规范文档

## 1. Git 工作流

### 1.1 分支策略

| 分支名 | 用途 | 说明 |
|--------|------|------|
| `main` | 主分支 | 稳定版本，仅用于发布 |
| `develop` | 开发分支 | 日常开发集成 |
| `feature/xxx` | 功能分支 | 新功能开发 |
| `bugfix/xxx` | 修复分支 | Bug 修复 |
| `hotfix/xxx` | 紧急修复 | 生产环境紧急修复 |

### 1.2 Commit 规范

采用 Conventional Commits 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**：
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具链相关

**示例**：
```
feat(tts): add GPT-SoVITS engine support

- Implement GPT-SoVITS adapter
- Add voice configuration UI
- Add unit tests

Closes #123
```

## 2. 前端开发规范 (React + TypeScript)

### 2.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件 | PascalCase | `ControlPanel.tsx` |
| 函数组件 | PascalCase | `function SubtitleWindow()` |
| Hooks | use + PascalCase | `useConfig()` |
| 普通函数 | camelCase | `formatDate()` |
| 常量 | UPPER_SNAKE_CASE | `MAX_LOG_LINES` |
| CSS 类 | kebab-case | `.btn-primary` |
| 接口/类型 | PascalCase (I 前缀可选) | `interface AppConfig` |

### 2.2 组件规范

**文件结构**：
```
components/
└── Button/
    ├── index.tsx          # 组件主文件
    ├── Button.tsx         # 组件实现
    ├── Button.test.tsx    # 测试文件
    └── styles.css         # 样式文件
```

**组件写法**：
```tsx
import React from 'react';

interface ButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
  children: React.ReactNode;
  onClick?: () => void;
  disabled?: boolean;
}

export const Button: React.FC<ButtonProps> = ({
  variant = 'primary',
  size = 'md',
  children,
  onClick,
  disabled = false,
}) => {
  return (
    <button
      className={`btn btn-${variant} btn-${size}`}
      onClick={onClick}
      disabled={disabled}
    >
      {children}
    </button>
  );
};
```

### 2.3 TypeScript 规范

- 优先使用 `interface` 定义对象类型
- 使用 `type` 定义联合类型、工具类型
- 避免使用 `any`，必要时用 `unknown`
- 函数必须定义返回类型

```typescript
// 推荐
interface User {
  id: string;
  name: string;
}

type Status = 'idle' | 'loading' | 'success' | 'error';

function getUser(id: string): Promise<User> {
  return fetch(`/api/users/${id}`).then(res => res.json());
}
```

### 2.4 状态管理 (Zustand)

```typescript
import { create } from 'zustand';

interface AppState {
  config: AppConfig;
  setConfig: (config: Partial<AppConfig>) => void;
  resetConfig: () => void;
}

export const useAppStore = create<AppState>((set) => ({
  config: defaultConfig,
  setConfig: (newConfig) =>
    set((state) => ({ config: { ...state.config, ...newConfig } })),
  resetConfig: () => set({ config: defaultConfig }),
}));
```

### 3.3 FastAPI 路由规范

```python
from fastapi import APIRouter, Depends, HTTPException
from pydantic import BaseModel

router = APIRouter(prefix="/api/tts", tags=["tts"])

class SynthesizeRequest(BaseModel):
    text: str
    voice_id: str
    speed: float = 1.0

class SynthesizeResponse(BaseModel):
    audio_url: str
    duration: float

@router.post("/synthesize", response_model=SynthesizeResponse)
async def synthesize_speech(
    request: SynthesizeRequest,
    tts_service: TTSService = Depends(get_tts_service),
):
    try:
        result = await tts_service.synthesize(
            text=request.text,
            voice_id=request.voice_id,
            speed=request.speed,
        )
        return SynthesizeResponse(
            audio_url=result.audio_url,
            duration=result.duration,
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
```

### 6.3 端口配置规范

**端口范围**：8500-8600

| 服务 | 默认端口 | 环境变量 | 说明 |
|------|----------|----------|------|
| API Server | 8500 | `API_PORT` | FastAPI 主服务 |
| Ollama Proxy | 8501 | `OLLAMA_PROXY_PORT` | Ollama 代理服务 |
| WebSocket | 8502 | `WS_PORT` | 实时通信服务 |
| Subtitle Service | 8503 | `SUBTITLE_PORT` | 字幕窗口通信 |
| TTS Service | 8504 | `TTS_PORT` | 语音合成服务 |
| Log Service | 8505 | `LOG_PORT` | 日志推送服务 |

**端口冲突处理**：
- 启动时自动检测端口占用
- 如被占用，自动尝试下一个可用端口
- 端口信息通过 IPC 传递给前端

## 7. 性能优化指南

### 7.1 前端优化

- 使用 React.memo 避免不必要重渲染
- 图片懒加载
- 虚拟列表处理长列表
- 路由懒加载
- 避免在 render 中创建新对象/函数

### 7.2 后端优化

- 使用异步 I/O (async/await)
- 数据库查询优化
- 缓存常用数据
- 响应压缩
- 连接池管理

## 8. 安全规范

- 永远不要提交 `.env` 文件到 Git
- API Key 等敏感信息使用环境变量
- 前端不要存储敏感信息
- 后端对所有输入进行验证
- 使用 HTTPS
- 实施 CORS 策略
- 日志中不要记录敏感信息
