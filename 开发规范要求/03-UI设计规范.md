# AIEffect - UI/UX 设计规范

## 1. 设计风格定位

### 1.1 整体风格
**现代极简科技风（AI 蓝紫风）**

- 核心特点：专业、可信、创新、未来感
- 适用场景：AI 工具、大模型产品、云服务
- 视觉感受：护眼、通透、高级感

---

## 2. 配色系统

### 2.1 核心配色逻辑

| 颜色类型 | 颜色值 | 用途说明 |
|----------|--------|----------|
| **主背景色** | `#f0f4ff` → `#e8f0ff` | 浅蓝/淡青渐变背景，护眼不刺眼 |
| **主色调 (紫蓝)** | `#6366f1` (Indigo) | 品牌色、视觉焦点、按钮、图标 |
| **主色调 (渐变)** | `#8b5cf6` → `#6366f1` | 紫蓝渐变，用于强调区域 |
| **辅助色 (青)** | `#06b6d4` (Cyan) | 次要强调、成功状态 |
| **强调色 (粉紫)** | `#ec4899` (Pink) | 高亮、危险、激活状态 |

### 2.2 中性色系统

| 颜色 | Hex | 用途 |
|------|-----|------|
| 黑色 | `#0f172a` | 主标题、重要文字 |
| 深灰 | `#334155` | 副标题、正文 |
| 中灰 | `#64748b` | 辅助文字、说明文字 |
| 浅灰 | `#94a3b8` | 占位符、禁用状态 |
| 极浅灰 | `#e2e8f0` | 分割线、边框 |
| 白色 | `#ffffff` | 卡片背景、次要按钮 |

### 2.3 功能色

| 状态 | 颜色 | Hex |
|------|------|-----|
| 成功 | 绿色 | `#10b981` |
| 警告 | 橙色 | `#f59e0b` |
| 错误 | 红色 | `#ef4444` |
| 信息 | 蓝色 | `#3b82f6` |

### 2.4 CSS 变量定义

```css
:root {

  --primary: #6366f1;
```

## 3. 组件设计规范

### 3.1 按钮 (Button)

#### 主按钮 (Primary)
- 背景：紫蓝渐变 `var(--gradient-primary)`
- 文字：白色 `#ffffff`
- 圆角：8px
- 阴影：`var(--shadow-md)`
- Hover：上移 2px + 发光阴影
- Active：下移 1px

```css
.btn-primary {
  background: var(--gradient-primary);
  color: white;
  border: none;
  padding: 10px 20px;
  border-radius: 8px;
  font-weight: 600;
  cursor: pointer;
  transition: all 0.2s ease;
  box-shadow: var(--shadow-md);
}

.btn-primary:hover {
  transform: translateY(-2px);
  box-shadow: var(--shadow-glow), var(--shadow-lg);
}
```

#### 次要按钮 (Secondary)
- 背景：白色 `#ffffff`
- 文字：主色 `var(--primary)`
- 边框：1px `var(--border)`
- 圆角：8px

#### 危险按钮 (Danger)
- 背景：`#ef4444`
- 文字：白色

#### 按钮尺寸
| 尺寸 | 高度 | 内边距 | 字体大小 |
|------|------|--------|----------|
| Small | 32px | 8px 16px | 13px |
| Default | 40px | 10px 20px | 14px |
| Large | 48px | 14px 28px | 16px |

### 3.2 输入框 (Input)

- 背景：白色 `#ffffff`
- 边框：1px `var(--border)`
- 圆角：8px
- 内边距：10px 14px
- Focus：边框变主色 + 发光阴影

```css
.input {
  width: 100%;
  padding: 10px 14px;
  border: 1px solid var(--border);
  border-radius: 8px;
  background: white;
  font-size: 14px;
  transition: all 0.2s ease;
}

.input:focus {
  outline: none;
  border-color: var(--primary);
  box-shadow: 0 0 0 3px rgba(99, 102, 241, 0.1);
}
```

### 3.3 卡片 (Card)

- 背景：白色 `#ffffff`
- 圆角：12px
- 阴影：`var(--shadow-sm)` 或 `var(--shadow)`
- 内边距：20px
- Hover：阴影加深（可选）

```css
.card {
  background: white;
  border-radius: 12px;
  box-shadow: var(--shadow);
  padding: 20px;
  border: 1px solid var(--border-light);
}
```

### 3.4 开关 (Switch)

- 开启状态：主色渐变 `var(--gradient-primary)`
- 关闭状态：浅灰 `#cbd5e1`
- 圆形滑块：白色

### 3.5 标签 (Tag/Badge)

| 类型 | 背景 | 文字 |
|------|------|------|
| 成功 | `#d1fae5` | `#065f46` |
| 警告 | `#fef3c7` | `#92400e` |
| 错误 | `#fee2e2` | `#991b1b` |
| 信息 | `#dbeafe` | `#1e40af` |
| 紫色 | `#ede9fe` | `#5b21b6` |

## 4. 页面布局规范

### 4.1 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  标题栏 (56px)                                            │
├──────────┬──────────────────────────────────────────────┤
│          │  页面头部 (80px)                              │
│          ├──────────────────────────────────────────────┤
│  侧边栏  │                                               │
│  (260px) │         主内容区域                            │
│          │                                               │
│          ├──────────────────────────────────────────────┤
│          │  底部状态栏 (72px)                            │
└──────────┴──────────────────────────────────────────────┘
```

### 4.2 侧边栏 (Sidebar)

- 宽度：260px
- 背景：白色
- 右侧边框：1px `var(--border)`
- Logo 区域高度：80px
- 导航项高度：48px

**导航项样式**：
- 默认：透明背景，深灰文字
- Hover：浅灰背景 `#f8fafc`
- 激活：紫蓝渐变背景 + 白色文字 + 左侧 4px 高亮条

### 4.3 主内容区域

- 背景：渐变背景 `var(--gradient-bg)`
- 内边距：24px
- 最大宽度：无限制（自适应）

**内容间距**：
- 卡片之间：20px
- 页面标题与内容：24px
- 表单元素之间：16px

### 4.4 栅格系统

采用 12 列栅格，基于 Tailwind CSS：

| 断点 | 宽度 | 列间距 |
|------|------|--------|
| sm | ≥ 640px | 20px |
| md | ≥ 768px | 20px |
| lg | ≥ 1024px | 24px |
| xl | ≥ 1280px | 24px |

## 5. 排版规范

### 5.1 字体家族

- 中文字体：`PingFang SC`, `Microsoft YaHei`, `Noto Sans SC`
- 英文字体：`Inter`, `-apple-system`, `BlinkMacSystemFont`, `Segoe UI`
- 代码字体：`Fira Code`, `Consolas`, `Monaco`

### 5.2 字体层级

| 层级 | 字体大小 | 字重 | 行高 | 用途 |
|------|----------|------|------|------|
| H1 | 32px | 700 | 1.3 | 页面大标题 |
| H2 | 24px | 600 | 1.4 | 区块标题 |
| H3 | 20px | 600 | 1.4 | 卡片标题 |
| Body | 14px | 400 | 1.6 | 正文 |
| Small | 12px | 400 | 1.5 | 辅助说明 |

### 5.3 文本颜色

- 主标题：`var(--text-primary)`
- 副标题/正文：`var(--text-secondary)`
- 辅助文字：`var(--text-muted)`
- 链接：`var(--primary)`

## 6. 图标与图形

### 6.1 图标库
- 使用：Lucide React / Heroicons / Ant Design Icons
- 风格：线性图标
- 尺寸：16px, 20px, 24px

### 6.2 装饰图形

**科技光点效果**：
- 白色光点 `rgba(255, 255, 255, 0.8)`
- 模糊效果 `blur(8px)`
- 随机分布
- 缓慢动画

**渐变背景装饰**：
- 紫蓝渐变圆形
- 高斯模糊
- 放置于页面角落作为背景装饰

## 7. 动效规范

### 7.1 过渡动画

| 属性 | 时长 | 缓动函数 |
|------|------|----------|
| 颜色 | 150ms | ease-out |
| 变换 | 200ms | ease-out |
| 阴影 | 200ms | ease-out |
| 透明度 | 150ms | ease-out |

### 7.2 交互反馈

- Hover：轻微上移 + 阴影加深
- Click：按压效果（scale 0.98）
- Loading：旋转加载动画 + 骨架屏

### 7.3 页面过渡

- 淡入淡出：`opacity 0.3s ease`
- 滑动进入：`transform 0.3s ease`

## 8. 各页面设计要点

### 8.1 控制面板
- 状态标签使用圆角胶囊样式
- 配置项分组放入卡片
- 重要按钮（启动服务）使用主按钮
- **端口显示**：显示当前使用的端口（8500-8600范围），冲突时显示实际分配的端口

### 8.2 字幕视觉
- 实时预览区域背景深色
- 预览窗口有悬浮效果
- 滑块控件样式统一

### 8.3 语音配置
- 标签页使用紫蓝激活状态
- 音量滑块使用主色
- 连接状态使用颜色标识

### 8.4 记忆管理
- 使用说明区域使用浅黄背景
- 输入框分组清晰
- 操作按钮排列整齐

### 8.5 运行日志
- 日志行使用斑马纹（浅灰交替）
- 日志级别使用颜色标识
- 时间戳使用等宽字体

## 9. 桌面应用 UI 规范

### 9.1 窗口控制

#### 标题栏
- 高度：32px（Windows）/ 40px（macOS）
- 背景：与侧边栏一致（白色）
- 左侧：应用图标 + 应用名称
- 右侧：窗口控制按钮

#### 窗口控制按钮（Windows）
```css
/* 最小化/最大化/关闭按钮 */
.window-control {
  width: 46px;
  height: 32px;
  display: flex;
  align-items: center;
  justify-content: center;
  transition: background 0.15s;
}

.window-control:hover {
  background: rgba(0, 0, 0, 0.05);
}

.window-control.close:hover {
  background: #e81123;
  color: white;
}
```

### 9.2 系统托盘

#### 托盘图标
- 尺寸：16x16px（Windows）/ 18x18px（macOS）
- 格式：ICO（Windows）/ ICNS（macOS）/ PNG（Linux）
- 状态指示：
  - 正常：彩色图标
  - 运行中：添加绿色指示点
  - 错误：添加红色指示点

#### 托盘菜单
```
┌─────────────────────────┐
│  AIEffect               │
├─────────────────────────┤
│  显示主窗口             │
│  启动服务          ✓    │
│  停止服务               │
├─────────────────────────┤
│  检查更新...            │
│  设置...                │
├─────────────────────────┤
│  退出                   │
└─────────────────────────┘
```

### 9.3 通知/Toast

#### 通知样式
- 位置：屏幕右下角（Windows/Linux）、右上角（macOS）
- 宽度：320px
- 圆角：12px
- 阴影：`var(--shadow-lg)`

```css
.notification {
  background: white;
  border-radius: 12px;
  box-shadow: var(--shadow-lg);
  padding: 16px;
  border-left: 4px solid var(--primary);
}

.notification.success {
  border-left-color: var(--success);
}

.notification.error {
  border-left-color: var(--error);
}
```

#### 通知类型
- **服务启动**："Ollama 代理服务已启动（端口：8501）"
- **端口冲突**："端口 8500 被占用，已自动切换到 8506"
- **更新可用**："发现新版本 v1.2.0"
- **错误提示**："连接失败，请检查配置"

### 9.4 安装向导界面

#### 安装步骤
1. **欢迎页**：应用 Logo + 应用名称 + 简介
2. **许可协议**：滚动文本 + 同意复选框
3. **安装路径**：路径选择 + 磁盘空间提示
4. **安装进度**：进度条 + 当前操作说明
5. **完成页**：安装成功提示 + 启动选项

#### 安装进度条
```css
.install-progress {
  height: 4px;
  background: var(--border);
  border-radius: 2px;
  overflow: hidden;
}

.install-progress-bar {
  height: 100%;
  background: var(--gradient-primary);
  transition: width 0.3s ease;
}
```

### 9.5 更新提示界面

#### 更新可用对话框
- 标题："发现新版本"
- 内容：
  - 当前版本 vs 新版本
  - 更新日志（可展开）
  - 文件大小
- 按钮："立即更新" / "稍后提醒" / "跳过此版本"

#### 下载进度
- 进度条 + 百分比
- 下载速度
- 预计剩余时间
- 取消按钮

## 10. 响应式设计

虽然是桌面应用，但支持窗口大小调整：
- 最小窗口尺寸：1000x700px
- 侧边栏最小宽度：220px
- 侧边栏最大宽度：320px
- 内容区域自适应
- 卡片在小窗口单列显示

### 断点设计
| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| 默认 | ≥ 1280px | 标准布局 |
| 中等 | 1024-1279px | 侧边栏收缩 |
| 紧凑 | < 1024px | 卡片单列 |
