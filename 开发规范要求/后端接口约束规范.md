# AIEffect - 后端接口约束规范

## 1. 概述

### 1.1 文档目的

本文档定义 AIEffect 后端 API 的接口规范，为前后端数据交互提供统一标准，确保接口的一致性、可维护性和可扩展性。

### 1.2 技术栈

- **后端框架**: FastAPI (Python)
- **API 协议**: RESTful API + WebSocket + SSE
- **数据格式**: JSON
- **认证方式**: API Key (可选)

**关于 SSE (Server-Sent Events)**:

- 用于实时推送场景（日志流、服务状态等）
- 前端通过 `EventSource` 连接，无需轮询
- 详见各模块 SSE 接口说明

### 1.3 端口分配

| 服务             | 端口  | 说明               |
| ---------------- | ----- | ------------------ |
| API Server       | 8501  | FastAPI 主服务     |
| Ollama Proxy     | 11434 | Ollama API 代理    |
| WebSocket        | 8502  | 实时通信服务       |
| Subtitle Service | 8503  | 字幕窗口通信       |
| TTS Service      | 8504  | 语音合成服务       |
| Log Service      | 8505  | 日志推送服务 (SSE) |

## 2. 通用规范

### 2.1 响应格式

所有 API 响应统一使用以下格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**字段说明**:

- `code`: 状态码 (参考 2.2 节)
- `message`: 状态描述
- `data`: 响应数据 (可为 null、对象或数组)

### 2.2 状态码定义

| 状态码 | 含义           | 说明                       |
| ------ | -------------- | -------------------------- |
| 200    | 成功           | 请求处理成功               |
| 400    | 请求参数错误   | 缺少必要参数或参数格式错误 |
| 401    | 未授权         | API Key 无效或缺失         |
| 403    | 禁止访问       | 权限不足                   |
| 404    | 资源不存在     | 请求的资源不存在           |
| 409    | 资源冲突       | 资源已存在或状态冲突       |
| 422    | 验证错误       | 请求数据验证失败           |
| 500    | 服务器内部错误 | 服务器处理异常             |
| 503    | 服务不可用     | 依赖服务不可用             |

### 2.3 分页规范

列表接口统一使用分页参数：

**请求参数**:

```json
{
  "page": 1,
  "page_size": 50
}
```

**响应格式**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "page_size": 50,
    "total_pages": 2
  }
}
```

### 2.4 时间格式

所有时间字段统一使用 ISO 8601 格式：

```
2024-01-15T10:30:00Z
```

## 3. 接口模块定义

### 3.1 配置管理模块 (Config)

#### 3.1.1 获取全局配置

```
GET /api/config
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "api": {
      "provider": "deepseek",
      "api_url": "https://api.deepseek.com/v1",
      "api_key": "sk-****",
      "model_name": "deepseek-chat"
    },
    "tts": {
      "enabled": true,
      "engine": "gpt_sovits",
      "volume": 0.8,
      "auto_play": true
    },
    "subtitle": {
      "font_color": "#ffffff",
      "background_color": "#0a0a0f",
      "opacity": 0.9,
      "font_size": 16,
      "typing_speed": 30
    },
    "memory": {
      "save_dir": "./data/memories",
      "trigger_threshold": 300,
      "compress_count": 50,
      "check_frequency": 30,
      "auto_compress": false
    },
    "ports": {
      "api": 8500,
      "ollama_proxy": 8501,
      "websocket": 8502,
      "subtitle": 8503,
      "tts": 8504,
      "log": 8505
    },
    "lan_enabled": false
  }
}
```

#### 3.1.2 更新全局配置

```
PUT /api/config
```

**请求体**:

```json
{
  "api": {
    "provider": "deepseek",
    "api_url": "https://api.deepseek.com/v1",
    "api_key": "sk-xxx",
    "model_name": "deepseek-chat"
```

**说明**: 支持部分更新，只更新提供的字段。

#### 3.1.3 重置配置为默认



#### 3.12.3 服务状态实时推送 (SSE)

```
GET /api/system/health/stream
```

**说明**: 使用 Server-Sent Events (SSE) 实时推送服务状态，前端无需轮询。

**⚠️ 重要提示**: 前端已实现 SSE 连接接收实时状态，请勿重复实现轮询检测，避免不必要的网络开销。

**SSE 数据格式**:

```
data: {"status": "healthy", "port": 8501, "timestamp": "2024-01-15T10:30:00Z"}
```

**推送频率**: 每 5 秒推送一次

**前端实现参考**:

```typescript
const es = new EventSource(`${API_BASE_URL}/api/system/health/stream`);
es.onmessage = (event) => {
  const data = JSON.parse(event.data);
  // data.status: "healthy" | "unhealthy"
  // data.port: 服务端口
  // data.timestamp: ISO 8601 时间戳
};
```

#### 3.13.5 导入角色文件（JSON）

```

```

用于接收外部角色卡/存档导出的 JSON，并进行字段归一化写入。

**请求体**:

```json
{
  "payload": {
    "Name": "Alice",
    "StringId": "save_001",
    "AIGeneratedPersonality": "温柔善良",

    "ConversationHistory": ["Player: 你好", "Alice: 你好呀"]
  }
}
```

**说明**:

- `payload` 原样落库到 `extra_data/metadata`，同时将已识别字段映射到标准字段：
  - `Name|name -> name`
  - `StringId|saveId|save_id -> save_id`
  - `AIGeneratedPersonality|CharacterDescription|aiSoul -> ai_soul`
  - `AssignedTTSVoice|aiVoice -> ai_voice`
  - `ConversationHistory -> conversation_messages`

## 8. 安全规范

### 8.1 API Key 管理

- 敏感配置 (如 API Key) 存储在本地加密数据库
- 接口返回时脱敏处理 (如 `sk-****`)

### 8.2 CORS 配置

- 开发环境: 允许所有来源
- 生产环境: 仅允许本地地址

### 8.3 请求限制

- 默认无速率限制 (本地应用)
- 代理服务可配置请求超时
