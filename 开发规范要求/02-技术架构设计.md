# AIEffect - 技术架构设计文档

## 1. 技术选型

### 1.1 整体技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端框架** | Electron + React + TypeScript | 跨平台桌面应用，现代前端框架 |
| **UI 组件库** | Ant Design / shadcn/ui | 现代化 UI 组件，支持自定义主题 |
| **样式方案** | Tailwind CSS + CSS Variables | 原子化 CSS + 自定义主题变量 |
| **后端框架** | FastAPI (Python) | 高性能异步 Web 框架 |
| **API 协议** | RESTful API + WebSocket (SSE) | 同步 + 实时推送 |
| **数据存储** | JSON 文件 + SQLite | 配置存储 + 轻量级数据 |
| **状态管理** | Zustand / Redux Toolkit | 前端状态管理 |
| **进程通信** | IPC (Electron) | 前后端通信 |

### 1.2 选型理由

#### 前端：Electron + React + TypeScript
- **Electron**：跨平台支持（Windows/macOS/Linux），丰富的桌面能力
- **React**：组件化开发，生态成熟，学习成本低
- **TypeScript**：类型安全，提升代码可维护性

#### 后端：FastAPI (Python)
- **高性能**：基于 Starlette 和 Pydantic，异步支持
- **自动文档**：Swagger UI / ReDoc 自动生成
- **Python 生态**：丰富的 AI/ML 库，便于集成语音合成等功能
- **类型提示**：完整的类型支持，减少错误

## 4. 前后端通信

### 4.1 IPC 通信 (Electron ↔ Python)

Electron 主进程作为桥梁，连接渲染进程和 Python 后端：

```typescript
// 主进程 (TypeScript)
import { ipcMain } from 'electron';
import { spawn } from 'child_process';
import * as net from 'net';

// 端口配置（8500-8600范围）
const PORT_CONFIG = {
  API: 8500,
  OLLAMA_PROXY: 8501,
  WEBSOCKET: 8502,
  SUBTITLE: 8503,
  TTS: 8504,
  LOG: 8505,
};

// 端口管理器 - 自动处理端口冲突
class PortManager {
  async findAvailablePort(preferredPort: number): Promise<number> {
    return new Promise((resolve, reject) => {
      const server = net.createServer();
      server.listen(preferredPort, '127.0.0.1', () => {
        const port = (server.address() as net.AddressInfo).port;
        server.close(() => resolve(port));
      });
      server.on('error', (err: any) => {
        if (err.code === 'EADDRINUSE') {
          this.findAvailablePort(preferredPort + 1).then(resolve).catch(reject);
        } else {
          reject(err);
        }
      });
    });
  }
}

// 启动 Python 后端并传递端口配置
async function startBackend() {
  const portManager = new PortManager();
  const ports = {
    API: await portManager.findAvailablePort(PORT_CONFIG.API),
    OLLAMA_PROXY: await portManager.findAvailablePort(PORT_CONFIG.OLLAMA_PROXY),
    WEBSOCKET: await portManager.findAvailablePort(PORT_CONFIG.WEBSOCKET),
    SUBTITLE: await portManager.findAvailablePort(PORT_CONFIG.SUBTITLE),
    TTS: await portManager.findAvailablePort(PORT_CONFIG.TTS),
    LOG: await portManager.findAvailablePort(PORT_CONFIG.LOG),
  };
  
  const pythonProcess = spawn('python', ['backend/src/main.py'], {
    env: {
      ...process.env,
      AIEFFECT_API_PORT: String(ports.API),
      AIEFFECT_OLLAMA_PROXY_PORT: String(ports.OLLAMA_PROXY),
      AIEFFECT_WS_PORT: String(ports.WEBSOCKET),
      AIEFFECT_SUBTITLE_PORT: String(ports.SUBTITLE),
      AIEFFECT_TTS_PORT: String(ports.TTS),
      AIEFFECT_LOG_PORT: String(ports.LOG),
    }
  });
  
  return ports;
}

// 转发 IPC 消息到 Python 后端
ipcMain.handle('api-call', async (event, endpoint, data) => {
  const apiPort = global.backendPorts?.API || 8500;
  const response = await fetch(`http://localhost:${apiPort}${endpoint}`, {
    method: 'POST',
    body: JSON.stringify(data)
  });
  return response.json();
});
```

### 4.2 RESTful API 设计

**端口配置说明**：所有服务端口均在 8500-8600 范围内，支持自动冲突检测和动态分配。

| 服务 | 默认端口 | 环境变量 | 描述 |
|------|----------|----------|------|
| API Server | 8500 | `AIEFFECT_API_PORT` | FastAPI 主服务 |
| Ollama Proxy | 8501 | `AIEFFECT_OLLAMA_PROXY_PORT` | Ollama API 代理 |
| WebSocket | 8502 | `AIEFFECT_WS_PORT` | 实时通信服务 |
| Subtitle Service | 8503 | `AIEFFECT_SUBTITLE_PORT` | 字幕窗口通信 |
| TTS Service | 8504 | `AIEFFECT_TTS_PORT` | 语音合成服务 |
| Log Service | 8505 | `AIEFFECT_LOG_PORT` | 日志推送服务 |

**API 端点**：

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/config` | 获取配置 |
| PUT | `/api/config` | 更新配置 |
| POST | `/api/proxy/start` | 启动代理服务 |
| POST | `/api/proxy/stop` | 停止代理服务 |
| POST | `/api/tts/synthesize` | 语音合成 |
| GET | `/api/subtitle/window` | 获取字幕窗口状态 |
| POST | `/api/subtitle/show` | 显示字幕 |
| GET | `/api/memory/characters` | 获取角色列表 |
| POST | `/api/memory/compress` | 压缩记忆 |
| GET | `/api/logs` | 获取日志列表 |
| GET | `/api/ports` | 获取当前使用的端口信息 |

### 4.3 WebSocket/SSE 事件

**事件类型**：
- `log:new` - 新日志
- `subtitle:update` - 字幕更新
- `service:status` - 服务状态变更
- `tts:complete` - 语音合成完成

## 5. 数据模型

### 5.1 配置模型 (Config)

```python
from pydantic import BaseModel
from typing import Optional

class APIConfig(BaseModel):
    provider: str = "custom"
    api_url: str = ""
    api_key: str = ""
    model_name: str = ""

class TTSConfig(BaseModel):
    enabled: bool = True
    engine: str = "gpt_sovits"
    volume: float = 0.8
    auto_play: bool = True

class SubtitleConfig(BaseModel):
    font_color: str = "#ffffff"
    background_color: str = "#0a0a0f"
    opacity: float = 0.9
    font_size: int = 16
    typing_speed: int = 30

class MemoryConfig(BaseModel):
    save_dir: str = "./data/memories"
    trigger_threshold: int = 300
    compress_count: int = 50
    check_frequency: int = 30
    auto_compress: bool = False

class PortConfig(BaseModel):
    """端口配置（8500-8600范围）"""
    api: int = 8500                    # FastAPI 主服务
    ollama_proxy: int = 8501           # Ollama 代理服务
    websocket: int = 8502              # WebSocket 服务
    subtitle: int = 8503               # 字幕服务
    tts: int = 8504                    # TTS 服务
    log: int = 8505                    # 日志服务
    
    class Config:
        env_prefix = "AIEFFECT_"

class AppConfig(BaseModel):
    api: APIConfig = APIConfig()
    tts: TTSConfig = TTSConfig()
    subtitle: SubtitleConfig = SubtitleConfig()
    memory: MemoryConfig = MemoryConfig()
    ports: PortConfig = PortConfig()    # 端口配置
    lan_enabled: bool = False
```

## 6. 部署与打包

### 6.1 开发环境

```bash
# 后端
cd backend
pip install -r requirements.txt
# 使用默认端口 8500-8505
python src/main.py

# 前端
cd frontend
npm install
npm run dev
```

### 6.2 桌面应用打包

#### 6.2.1 打包流程

```bash
# 1. 构建后端（使用 Nuitka）
cd backend
python -m nuitka --standalone --output-dir=dist src/main.py

# 2. 构建前端
cd frontend
npm run build

# 3. 打包 Electron 应用
# Windows
npm run build:win

# macOS
npm run build:mac

# Linux
npm run build:linux
```

#### 6.2.2 打包配置

使用 electron-builder 打包，支持以下格式：

| 平台 | 格式 | 说明 |
|------|------|------|
| **Windows** | NSIS (.exe) | 安装程序 |
| | Portable (.exe) | 便携版 |
| **macOS** | DMG | 磁盘映像 |
| | ZIP | 压缩包 |
| **Linux** | AppImage | 通用格式 |
| | DEB | Debian/Ubuntu |

#### 6.2.3 端口冲突处理

打包后的应用会自动处理端口冲突：
1. 启动时检测默认端口（8500-8505）是否可用
2. 如被占用，自动尝试下一个可用端口（最多到8600）
3. 将实际使用的端口传递给后端服务
4. 前端通过 IPC 获取实际端口进行通信

使用 electron-updater 实现自动更新：
- 更新服务器：GitHub Releases
- 支持增量更新
- 支持 Windows、macOS、Linux
