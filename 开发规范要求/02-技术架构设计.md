# AIEffect - 技术架构设计文档

## 1. 技术选型

### 1.1 整体技术栈

| 层级          | 技术选型                      | 说明                           |
| ------------- | ----------------------------- | ------------------------------ |
| **前端框架**  | React 18 + TypeScript + Vite  | 现代化前端构建工具，快速 HMR   |
| **UI 组件库** | Tailwind CSS + Lucide Icons   | 原子化 CSS + 图标库            |
| **状态管理**  | Zustand                       | 轻量级状态管理                 |
| **可视化**    | PixiJS v8                     | 2D 图形渲染引擎                |
| **后端框架**  | FastAPI (Python)              | 高性能异步 Web 框架            |
| **API 协议**  | RESTful API + WebSocket       | 同步 + 实时双向通信            |
| **数据存储**  | SQLite + JSON 文件            | 轻量级数据存储                 |
| **部署方式**  | Docker + Docker Compose       | 容器化部署                     |
| **依赖管理**  | Poetry (Python) / npm (Node)  | 现代化依赖管理                 |

### 1.2 选型理由

#### 前端：React + TypeScript + Vite

- **Vite**：极速的开发和构建体验，原生 ESM 支持
- **React**：组件化开发，生态成熟，学习成本低
- **TypeScript**：类型安全，提升代码可维护性
- **PixiJS**：高性能 2D 渲染，适合 Agents 工作室可视化

#### 后端：FastAPI + 微服务架构

- **高性能**：基于 Starlette 和 Pydantic，异步支持
- **自动文档**：Swagger UI / ReDoc 自动生成
- **微服务**：服务独立部署、扩展，故障隔离
- **Python 生态**：丰富的 AI/ML 库，便于集成语音合成等功能

#### 部署：Docker Compose

- **一致性**：开发、测试、生产环境一致
- **便捷性**：一键启动所有服务
- **可扩展**：易于添加新服务

## 3. 核心模块设计

### 3.1 API Gateway 模块

**职责**：
- 统一 API 入口，对外暴露 RESTful 接口
- 路由转发到内部服务或外部 API
- 请求认证与限流
- 业务逻辑处理

**核心功能**：
- Ollama/OpenAI API 代理
- TTS 语音合成接口
- 字幕服务接口
- 记忆管理接口
- Agents 管理接口
- 文件上传/下载

**技术实现**：
```python
# services/api_gateway/src/api_gateway/main.py
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI(title="AIEffect API Gateway")

# CORS 配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 注册路由
app.include_router(proxy.router, prefix="/api/proxy", tags=["proxy"])
app.include_router(tts.router, prefix="/api/tts", tags=["tts"])
app.include_router(agents.router, prefix="/api/agents", tags=["agents"])
```

### 3.2 WebSocket 服务模块

**职责**：
- 管理客户端长连接
- 实时消息推送
- 会话状态维护
- 多房间/频道支持

**核心功能**：
- 连接管理（连接、断开、心跳）
- 消息路由（按类型分发到不同处理器）
- 会话管理（创建、销毁、状态同步）
- 广播/组播消息

**技术实现**：
```python
# services/websocket/src/websocket_service/main.py
from fastapi import FastAPI, WebSocket
from websocket_service.connection.manager import ConnectionManager

app = FastAPI()
manager = ConnectionManager()

@app.websocket("/ws/{client_id}")
async def websocket_endpoint(websocket: WebSocket, client_id: str):
    await manager.connect(websocket, client_id)
    try:
        while True:
            data = await websocket.receive_json()
            await manager.handle_message(client_id, data)
    except WebSocketDisconnect:
        await manager.disconnect(client_id)
```

### 3.3 日志服务模块

**职责**：
- 接收各服务的日志数据
- 结构化存储到 SQLite
- 提供日志查询接口
- 实时日志推送 (SSE)
- 告警规则引擎

**核心功能**：
- 日志写入接口
- 日志查询接口（支持过滤、分页）
- SSE 实时推送
- 告警配置与触发

**技术实现**：
```python
# services/log/src/main.py
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import aiosqlite

app = FastAPI()

@app.post("/api/logs")
async def write_log(log: LogEntry):
    """接收日志写入"""
    async with aiosqlite.connect(DB_PATH) as db:
        await db.execute(
            "INSERT INTO logs (level, module, message, timestamp) VALUES (?, ?, ?, ?)",
            (log.level, log.module, log.message, log.timestamp)
        )
        await db.commit()

@app.get("/api/logs/stream")
async def stream_logs():
    """SSE 实时日志推送"""
    async def event_generator():
        while True:
            log = await get_latest_log()
            yield f"data: {log.json()}\n\n"
    return StreamingResponse(event_generator(), media_type="text/event-stream")
```

## 4. 服务间通信

### 4.1 通信方式概览

| 场景                     | 通信方式       | 说明                           |
| ------------------------ | -------------- | ------------------------------ |
| 前端 → API Gateway       | HTTP REST      | 标准 RESTful API               |
| 前端 → WebSocket         | WebSocket      | 实时双向通信                   |
| 前端 → Log Service       | HTTP + SSE     | 日志查询 + 实时推送            |
| API Gateway → Log Service| HTTP           | 内部服务调用                   |
| 服务 → Log Service       | HTTP           | 日志上报                       |

### 4.2 端口配置（AIEffect 专用区间 8500-8505）

| 端口 | 服务             | 环境变量              | 说明                    |
| ---- | ---------------- | --------------------- | ----------------------- |
| 8500 | Frontend         | `FRONTEND_PORT`       | 前端服务(Vite)          |
| 8501 | API Gateway      | `API_GATEWAY_PORT`    | FastAPI 主服务          |
| 8502 | WebSocket Service| `WEBSOCKET_PORT`      | WebSocket 服务          |
| 8503 | （保留）         | -                     | 字幕服务（未来）        |
| 8504 | （保留）         | -                     | TTS 服务（未来）        |
| 8505 | Log Service      | `LOG_SERVICE_PORT`    | 日志服务                |

### 4.3 API 设计规范

**RESTful API 端点**：

| 方法 | 路径                          | 描述                      |
| ---- | ----------------------------- | ------------------------- |
| GET  | `/api/health`                 | 健康检查                  |
| GET  | `/api/config`                 | 获取配置                  |
| PUT  | `/api/config`                 | 更新配置                  |
| POST | `/api/proxy/chat/completions`| OpenAI 兼容对话接口       |
| POST | `/api/tts/synthesize`         | 语音合成                  |
| GET  | `/api/agents`                 | 获取 Agents 列表          |
| POST | `/api/agents`                 | 创建 Agent                |
| GET  | `/api/logs`                   | 获取日志列表              |
| GET  | `/api/logs/stream`            | SSE 实时日志流            |

**WebSocket 消息格式**：
```json
{
  "type": "message_type",
  "payload": {},
  "timestamp": "2024-01-01T00:00:00Z",
  "client_id": "uuid"
```

## 5. 数据模型

### 5.1 配置模型

```python
# shared/python/shared_core/schemas.py
from pydantic import BaseModel, Field
from typing import Optional, List

class APIConfig(BaseModel):
    provider: str = "ollama"
    api_url: str = "http://localhost:11434"
    api_key: Optional[str] = None
    model_name: str = "llama2"

class TTSConfig(BaseModel):
    enabled: bool = True
    engine: str = "gpt_sovits"
    volume: float = 0.8
    auto_play: bool = True

class SubtitleConfig(BaseModel):
    font_color: str = "#ffffff"
    background_color: str = "#0a0a0f"
    opacity: float = 0.9
    font_size: int = 16
    typing_speed: int = 30

class MemoryConfig(BaseModel):
    save_dir: str = "./data/memories"
    trigger_threshold: int = 300
    compress_count: int = 50
    check_frequency: int = 30
    auto_compress: bool = False

class AppConfig(BaseModel):
    api: APIConfig = APIConfig()
    tts: TTSConfig = TTSConfig()
    subtitle: SubtitleConfig = SubtitleConfig()
    memory: MemoryConfig = MemoryConfig()
```

## 6. 部署架构

### 6.1 Docker Compose 配置

```yaml
# docker-compose.yml
version: '3.8'

services:
  frontend:
    build: ./frontend
    ports:
      - "${FRONTEND_PORT:-8500}:8500"
    environment:
      - VITE_API_URL=http://localhost:8501
      - VITE_WS_URL=ws://localhost:8502
    depends_on:
      - api_gateway
      - websocket

  api_gateway:
    build: ./services/api_gateway
    ports:
      - "${API_GATEWAY_PORT:-8501}:8501"
    environment:
      - LOG_SERVICE_URL=http://log_service:8505
      - WEBSOCKET_URL=ws://websocket:8502
    volumes:
      - ./data:/app/data
    depends_on:
      - log_service

  websocket:
    build: ./services/websocket
    ports:
      - "${WEBSOCKET_PORT:-8502}:8502"
    environment:
      - API_GATEWAY_URL=http://api_gateway:8501

  log_service:
    build: ./services/log
    ports:
      - "${LOG_SERVICE_PORT:-8505}:8505"
    volumes:
      - ./data/logs:/app/data
```

### 6.2 环境变量配置

```bash
# .env
```

## 8. 性能优化

### 8.1 前端优化

- 使用 React.memo 避免不必要重渲染
- PixiJS 画布使用对象池减少 GC
- 路由懒加载
- 图片懒加载

### 8.2 后端优化

- 使用异步 I/O (async/await)
- 数据库连接池
- 响应压缩 (gzip)
- 缓存常用数据

### 8.3 部署优化

- Docker 镜像多阶段构建
- 使用 .dockerignore 减少构建上下文
- 服务健康检查
- 自动重启策略

```python
@app.get("/health")
async def health_check():
```

## 10. 安全规范

- 敏感信息使用环境变量管理
- 永远不要提交 `.env` 文件到 Git
- API Key 等敏感信息不在前端暴露
- 后端对所有输入进行验证
- 使用 CORS 限制跨域访问
- 日志中不记录敏感信息
